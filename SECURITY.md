# Security policy

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Report them privately, either:

- through [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) on this repository ("Report a vulnerability"), or
- by email to [plemaistre@racine.ai](mailto:plemaistre@racine.ai).

Include what you can: affected component, reproduction steps, impact, and any suggested fix.

## What to expect

- We acknowledge reports within 72 hours.
- We keep you informed while we investigate and prepare a fix.
- We credit reporters in the release notes, unless you prefer otherwise.

## Supported versions

Security fixes are applied to the latest release and the `main` branch. Older releases are not
maintained; please upgrade to the latest version.
